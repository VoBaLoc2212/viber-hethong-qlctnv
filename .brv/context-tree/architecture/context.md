# Domain: architecture

## Purpose
Capture high-level software architecture and structural decisions for the financial management platform.

## Scope
Included in this domain:
- System components and boundaries
- Data flow and architecture style
- Cross-cutting concerns such as auth and audit
- Core technical stack decisions

Excluded from this domain:
- Team process rituals
- Detailed per-endpoint payload contracts

## Ownership
Engineering

## Usage
Use this domain to understand how the platform is structured before diving into module-level topics.
